Quel était le problème ?

Mon ami Xavier le cuisinier avait cassé sa caisse enregistreuse.

Il me fallait l'aider en lui créant un programme permettant à ses serveurs de :

  • - Ecrire ce qu'a consommé le client
  • - Editer un joli ticket de caisse pour ensuite l'imprimer



Comment s'est déroulée la conception ?

Il m'a fallu réfléchir dans un premier temps à ce dont j'avais besoin.

  • Récupérer des données, en l'occurence :
    • - Le type de plat
    • - La quantité
    • - Le prix unitaire du plat

  • Afficher une facture
  • Afficher les données récupérées et les mettre en forme
  • Calculer et afficher les prix de chaque plat

  • Calculer et afficher le prix total :
    • - Avant TVA
    • - Après TVA

Quid du codage ?

Le codage s'est découpé en plusieurs étapes :

Création de tableaux :

Déclaration de 3 tableaux

  • String [] articles pour les plats
  • int [] quantities pour les quantités
  • double [] prices pour les prix unitaires

Création d'une boucle pour l'enregistrement des données

Le choix s'est porté sur une boucle "do while" afin de poser une première question ("Quelle est la commande n°1").
La boucle continue tant que la réponse n'est pas un champ vide.
Enfin un compteur tourne avec la boucle pour compter les plats.

Affichage du ticket de caisse

Cette fois-ci c'est une boucle while que j'utilise tant que mon tableau de plats n'est pas vide.

  • while (articles[counter]!=null){
    • double linePrices = quantities[counter]*prices[counter];
    • totalPrice = totalPrice + linePrices;
    • System.out.printf("%2s x %10s (%6.2f€) %10.2f€, quantities[counter],
      • articles[counter], prices[counter], linePrices);
    • System.out.println();
    • counter++;
  • }

La variable double "linePrices" me permet d'enregsitrer les plats par leurs quantités,
tandis que totalPrice les incrémente.

Il ne me reste plus qu'à afficher la facture totale (sans tva puis avec tva).
J'ai choisi un System.out.printf pour formater mes données.
